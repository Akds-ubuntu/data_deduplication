# Data Deduplication Framework (DDF)

**Data Deduplication Framework** — это высокопроизводительная программная среда для поиска и удаления дубликатов в сверхбольших текстовых датасетах и массивах исходного кода. Система реализует гибридный двухэтапный пайплайн, позволяющий эффективно обходить вычислительную сложность $O(N^2)$.

##  Архитектура системы

Процесс дедупликации разделен на два независимых этапа, что обеспечивает баланс между скоростью обработки и точностью итоговой кластеризации.

### Этап 1: Генерация кандидатов (Candidate Generation)
Цель этапа — быстро отсеять заведомо уникальные документы и выделить пары «подозрительных» кандидатов.
1.  **Токенизация:** Формирование перекрывающихся N-грамм (shingles) на уровне символов или слов.
2.  **Хеширование:** Применение алгоритма `xxhash` для получения быстрых 64-битных отпечатков.
3.  **Стратегии сигнатур:** * **MinHash:** Создание компактных сигнатур через массив пермутаций.
    * **SimHash:** Вычисление весовых хэшей для оценки косинусного сходства (опционально).
    * **Semantic:** Прямая векторная репрезентация (пропускает этап LSH).
4.  **LSH (Locality-Sensitive Hashing):** Нарезка сигнатур на полосы (bands) и распределение по корзинам (buckets). При использовании алгоритма `minhashLSH`, документы, попавшие в один бакет, становятся «кандидатами» на дубликаты. В режимах `simhash` и `semantic` этот этап может быть заменен прямым сравнением по порогу.

### Этап 2: Точная верификация и кластеризация (Verification & Clustering)
Использование точного режима значительно повышает качество очистки данных ценой дополнительного времени на вычисления.
5.  **Фильтрация кандидатов:** Пары документов, отобранные на первом этапе, проходят строгую проверку. Поддерживается семантическая верификация через эмбеддинги (`FastEmbed`).
6.  **Кластеризация (Union-Find):** Подтвержденные пары объединяются в непересекающиеся множества (кластеры) с помощью структуры данных Disjoint-Set.

## Основные компоненты системы

| Модуль | Описание |
| :--- | :--- |
| `dedup_tool/cli.py` | Входная точка. Интерфейс командной строки на базе `Typer` с визуализацией через `Rich`. |
| `core/strategy.py` | Базовый абстрактный класс, определяющий интерфейс для всех алгоритмов дедупликации. |
| `core/strategyregistry.py` | Фабрика стратегий для динамической подгрузки нужного алгоритма из конфигурации. |
| `core/minhash.py` | Реализация логики MinHash и эффективного поиска соседей через LSH. |
| `core/semantic.py` | Модуль глубокой верификации на базе семантических эмбеддингов. |
| `utils/embeded.py` | Векторизованная генерация сигнатур с использованием `NumPy` для минимизации задержек. |
| `utils/union_find.py` | Оптимизированная структура данных для формирования итоговых кластеров. |

## Конфигурация (`config.yaml`)

Настройка параметров работы системы и выбор алгоритмов производится через файл конфигурации.

```yaml
# Основные настройки
algorithm: minhash       # Доступные: minhash, simhash, semantic
text_field: "func"       # Ключ в JSONL, содержащий целевой текст
input_path: "CodeXGLUE_Test.jsonl"
output_path: "results.json"
debug: false # Включение отладочных логов

# Параметры MinHash + LSH
num_perm: 512                   # Количество независимых хеш-функций
ngram_size: 3                   # Размер n-граммы (символьный шингл)
bands: 128                      # Количество полос в LSH (влияет на Recall)
threshold: 0.4                  # Порог сходства Жаккара
use_semantic_lsh: False         # Доп проверка симатнтики
threshold_semantic_lsh: 0.8    # Порог схожесть в семантическом плане

# Параметры SimHash (опционально)
# threshold: 3           # Максимальное расстояние Хэмминга (0-64)

# Параметры Semantic
model_name: "BAAI/bge-small-en-v1.5" # Модель для анализа
